Responsive Portfolio Part 4 - Bootstrap Carousel | Exercise

  • Due Wednesday by 11pm
  • Points 100
  • Submitting a website url
  • Available until Jun 28 at 11pm

*Note: It is imperative that all of your coursework throughout your time here at F.I.R.S.T. be saved on an external hard/flash drive from now through graduation. You will be referencing your work again in the future to be added to your portfolio, so we expect you not to delete any of your work, including working files.*

 

Today, we're going to introduce you to your first web framework: Bootstrap.  Bootstrap is a popular responsive web framework that also comes with a TON of components that you could add to your portfolio layout.  We could easily spend weeks on Bootstrap, so we're only going to give you a taste of it here, and make sure you know enough to dive into the rest of Bootstrap on your own, if you so choose.

We're going to focus on one of the more popular components that Bootstrap has to offer: the Carousel!  A Carousel is kind of like a slideshow, but much flashier.  Carousels and slideshows tend to be one of the most complex things to design in front-end web development, which is one of the reasons it's so nice to have access to a pre-built one in Bootstrap.

 

 

TECHNOLOGY WE'LL BE USING:

  • Visual Studio Code
  • GitHub Desktop App

 

HOW TO COMPLETE:

Assuming you've completed Part 3, follow along with your instructor's demo and/or do the steps described below.

 

Prepping our HTML

In the previous assignment, we had you populate your favorite gallery with your own images; we really don't need all of those galleries on your Portfolio, so get rid of all the HTML for the other coffee galleries before we continue forward.  (You can leave the CSS; that'll make it easier to add any of those other galleries back into the HTML if you really want or need them)

Now, you should only have ONE gallery left on the page.  Above the "gallery" <section> is our "carousel_container" <section>, and that's where we're going to be working today.

 

Including Bootstrap

Our last bit of prepwork involves pulling the Bootstrap library into our webpage, as described in their Getting Started page Links to an external site..  To do that, we need to add one <link> element (for the CSS) and one <script> element (for the JavaScript) in the HTML.

  • In index.html
    • include the <link> element in your <head> tag, just beneath the <title> tags 
      • <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0-alpha3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-KK94CHFLLe+nY2dmCWGMq91rCGa5gtU4mk92HdvYe+M/SXH301p5ILy+dN9+nJOZ" crossorigin="anonymous">
    • include the <script> element in your <body> tag, at the very end, right before the closing </body> tag 
      • <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0-alpha3/dist/js/bootstrap.bundle.min.js" integrity="sha384-ENjdO4Dr2bkBIFxQpeoTz1HIcje39Wm4jDKdf19U8gI4ddQ3GYNS7NTKfAdVQSZe" crossorigin="anonymous"></script>

If you put the <link> and <script> elements in the right places, then you should be set to use the Bootstrap framework!

NOTE: technically, you don't NEED both the <link> and the <script> elements.  The <link> element loads up the CSS part of Bootstrap, and the <script> element loads up the JavaScript part.  If all you need from Bootstrap is styling, you probably don't need the JavaScript <script> element.  However, if you plan to add anything from Bootstrap with interactivity (i.e. a carousel), you'll probably need that <script> too.

 

Adding a carousel

Bootstrap has MANY versions of its Carousel component.  Your assignment this time is relatively simple:

  • pick a version of the Carousel that you like from the Bootstrap Components library: https://getbootstrap.com/docs/5.3/components/carousel/ Links to an external site.
  • put the HTML code in your page inside the new "carousel_container" <section> 
  • put your own images inside the Carousel component!
    • HOWEVER, note that generally speaking, landscape-orientation images are going to work better with carousels than portrait-orientation images

You may find that if you use images of different sizes, you're run into some of the same issues we encountered in our galleries yesterday.  To fix those, you'll likely have to set "height" or "max-height" properties on the images to make them consistently sized (either with custom CSS, or using the Bootstrap sizing utility classes Links to an external site.), and possibly strip out some of the special Bootstrap classes (like "w-100", which tells the image to have width: 100%;)  Also look back at our example code from Part 3, and see if you can apply anything we learned to help you out here.

This assignment is deliberately a bit more freeform, to allow you folks to spread your wings, so to speak.  Of course, that also means you could potentially "fall", but if you get stuck, you can always reach out to your instructor for advice.  Our goal for this assignment is not perfection, but mostly just to get a functioning slideshow on your site.
